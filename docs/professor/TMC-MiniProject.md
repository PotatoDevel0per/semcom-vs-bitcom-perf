## 프로젝트 유형 (택 1)

**(a) 성능 비교형**
두 개 이상의 알고리즘·프로토콜·소프트웨어·시스템을 수학적 분석, 컴퓨터 시뮬레이션, 또는 실측(real measurement)으로 비교.
신규 설계도 환영하나 기존 대비 성능 비교가 반드시 포함되어야 함.

**(b) 서베이 논문형**
모바일 컴퓨팅·네트워킹의 신흥 주제에 대한 서베이 작성.
정성적/정량적 비교 및 자체 분석이 반드시 포함되어야 함.

> 모든 주제는 교수님 승인 필요.

## 2. 보고서 요건

### 성능 비교형 보고서 구성
title / authors / abstract / keywords / introduction / related works / main sections / performance evaluation / conclusions / references
- **참고문헌 10개 이상**, 모두 본문에서 최소 1회 인용

### 서베이 논문형 보고서 구성
title / authors / abstract / keywords / introduction / main sections / qualitative·quantitative comparison / challenging issues / conclusions / references
- **참고문헌 20개 이상**, 모두 본문에서 최소 1회 인용
- 권장 분량: **8~15페이지 이상** (IEEE 형식 기준)
