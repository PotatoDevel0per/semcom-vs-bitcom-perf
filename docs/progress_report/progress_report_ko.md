## 요약 (Abstract)

본 중간 보고서는 태스크 지향적 모바일 컴퓨팅 환경에서 Semantic 통신과 원시 비트(JPEG 압축) 전송 간의 성능 비교 연구의 현재까지 진행된 작업을 요약한다. CIFAR-10 이미지 분류를 하위 태스크로 하는 전체 시뮬레이션 파이프라인을 구축하였다. 기준선(baseline)으로 사용된 ResNet-18 분류기는 테스트 정확도 93.51%를 달성하였다. 잠재 차원(latent dimension) 64, 128, 256의 Semantic 인코더를 종단 간(end-to-end)으로 학습하고 AWGN 및 패킷 손실 채널 조건에서 평가하였다. 초기 결과에 따르면, Semantic 방식(dim=64, 64바이트)은 SNR=10 dB에서 91.94%의 정확도를 달성하였으며, 이는 무손실 기준선 대비 대역폭의 2.1%만을 사용하면서 JPEG 압축 대비 비트당 정확도(Accuracy per Bit) 기준으로 약 15배 우수한 성능이다.

## 1. 서론 (Introduction)

AI 기반 모바일 애플리케이션의 확산은 대역폭이 제한된 무선 채널에서 추론 태스크를 위한 효율적인 데이터 전송을 요구한다. JPEG와 같은 소스 코딩을 적용한 원시 비트 전송 방식은 하위 태스크와 무관하게 픽셀 수준의 정보를 전송하여 제한된 채널 자원을 비효율적으로 사용한다. Semantic 통신은 태스크 관련 특징만을 추출하여 전송함으로써 페이로드를 줄이면서도 태스크 성능을 유지한다 [1][2].

본 프로젝트는 현실적인 무선 채널 조건에서 Semantic 통신과 JPEG 기반 원시 비트 전송을 정량적으로 비교하며, Top-1 분류 정확도, 비트당 정확도(Accuracy per Bit, ApB), 채널 열화(SNR 변화 및 패킷 손실)에 대한 강건성의 세 가지 지표에 초점을 맞춘다.

---

## 2. 문제 정의 (Problem Statement)

대역폭이 제한된 무선 채널에서 송신측은 수신측이 이미지 분류를 올바르게 수행할 수 있도록 이미지 $x$의 표현을 전송해야 한다. 핵심 과제는 전송 크기(바이트)와 태스크 정확도 간의 균형이다. 전송 예산 $B$(바이트)에 대해 두 방식을 비교한다:

- **기준선(Baseline):** 품질 $q$의 JPEG 압축 방식으로, 이미지 $x$를 약 $B$ 바이트로 압축하여 전송 후 복원하고 분류기에 입력한다.
- **Semantic 방식:** 학습된 인코더 $f_\phi(x) \in \mathbb{R}^d$가 $d$차원 잠재 벡터를 추출하고, 8-bit 스칼라 양자화를 통해 $d$ 바이트로 양자화하여 잡음 채널을 통해 전송한 후 경량 헤드 $g_\psi$로 분류한다.

채널은 SNR이 변화하는 AWGN, Rayleigh 페이딩, 또는 블록 단위 패킷 손실로 모델링된다.

## 3. 완료된 작업 (Work Completed)

### 3.1 시뮬레이션 환경

전체 실험 파이프라인을 Python/PyTorch로 구현하고 GitHub에서 버전 관리 중이다. 시스템은 Apple MPS(M 시리즈 GPU)에서 실행된다. 주요 구성 요소:

- **데이터셋:** CIFAR-10 (학습 50,000개 / 테스트 10,000개, 32×32 RGB)
- **채널 시뮬레이터:** AWGN, Rayleigh 페이딩, 블록 단위 패킷 손실
- **양자화기:** 8-bit 균일 스칼라 양자화
- **기준선:** PIL을 통한 JPEG 압축 (품질 q ∈ {1, 3, 5, 10, 20, 40, 60, 80, 95})

### 3.2 기준선: ResNet-18 분류기

ResNet-18 분류기를 CIFAR-10에 맞게 수정하고(3×3 첫 번째 conv, stride 1, max-pooling 제거), SGD와 코사인 어닐링(lr=0.1, weight decay=5×10⁻⁴, momentum=0.9)을 사용하여 30 에폭 학습하였다. 모델은 **테스트 정확도 93.51%**를 달성하였으며, 이는 무손실 전송의 상한선 기준으로 사용된다.

### 3.3 Semantic 인코더 학습

잠재 차원 ∈ {64, 128, 256}의 SemanticEncoder 모델 3종을 동일한 ResNet-18 백본과 선형 투영 헤드를 사용하여 종단 간 학습하고, 2층 MLP 분류 헤드와 결합하였다. Adam(lr=1×10⁻³, 코사인 어닐링, 30 에폭)으로 학습하였다. 결과:

| 잠재 차원 | 전송 크기 | 최고 테스트 정확도 |
|:---------:|:---------:|:-----------------:|
| 64        | 64 B      | 92.52%            |
| 128       | 128 B     | 92.25%            |
| 256       | 256 B     | 92.42%            |

### 3.4 기준선 평가 (JPEG)

9개 품질 수준에서 JPEG 압축을 평가하였다. ResNet-18 분류기(§3.2)를 재학습 없이 복원된 이미지에 직접 적용하였다.

| 방식 | 평균 바이트 | Top-1 정확도 | Acc/Bit (×10⁻⁴) |
|:-----|:-----------:|:------------:|:----------------:|
| Raw (무손실) | 3,072 B | 93.51% | 0.38          |
| JPEG q=10  | 699.5 B   | 42.47%       | 0.76             |
| JPEG q=40  | 801.3 B   | 72.63%       | 1.13             |
| JPEG q=60  | 853.8 B   | 78.83%       | 1.15             |
| JPEG q=80  | 960.6 B   | 85.36%       | 1.11             |
| JPEG q=95  | 1,300.8 B | 91.70%       | 0.88             |

JPEG는 q=60에서 비트당 정확도가 최고치(1.15×10⁻⁴)에 도달하지만, ~79% 정확도를 위해 850 바이트 이상이 필요하다.

### 3.5 Semantic 평가

Semantic 인코더를 AWGN(SNR ∈ {0, 5, 10, 20} dB)과 패킷 손실(손실률 ∈ {1%, 5%, 10%}) 조건에서 평가하였다.

**표 3. AWGN 채널 조건에서의 Semantic 정확도 (CIFAR-10 테스트 세트)**

| 잠재 차원 | SNR=0 dB | SNR=5 dB | SNR=10 dB | SNR=20 dB |
|:---------:|:--------:|:--------:|:---------:|:---------:|
| 64        | 88.67%   | 91.32%   | 91.94%    | 92.52%    |
| 128       | 90.49%   | 91.92%   | 92.14%    | 92.40%    |
| 256       | 91.43%   | 92.16%   | 92.16%    | 92.37%    |

**표 4. 패킷 손실 조건에서의 Semantic 정확도**

| 잠재 차원 | 손실률=1% | 손실률=5% | 손실률=10% |
|:---------:|:---------:|:---------:|:----------:|
| 64        | 92.46%    | 92.24%    | 92.32%     |
| 128       | 92.21%    | 92.32%    | 92.18%     |
| 256       | 92.40%    | 92.41%    | 92.45%     |

**표 5. SNR=10 dB에서의 통신 효율 비교**

| 방식 | 바이트 | Top-1 정확도 | Acc/Bit (×10⁻⁴) |
|:-----|:------:|:------------:|:----------------:|
| JPEG q=60 (최고 ApB) | 853.8 | 78.83% | 1.15      |
| JPEG q=95        | 1,300.8 | 91.70%  | 0.88             |
| **Semantic dim=64**  | **64** | **91.94%** | **17.96**    |
| Semantic dim=128 | 128    | 92.14%      | 8.99             |
| Semantic dim=256 | 256    | 92.16%      | 4.50             |

## 4. 초기 연구 결과 (Preliminary Findings)

**(1) 획기적인 대역폭 절감:** Semantic dim=64는 단 **64 바이트**만으로 91.94% 정확도를 달성한다. 이는 무손실 기준선(3,072 바이트)의 2.1%에 불과하며, JPEG는 동등한 정확도(91.70%)를 달성하기 위해 1,301 바이트가 필요하다.

**(2) 통신 효율성:** Semantic dim=64의 비트당 정확도는 17.96×10⁻⁴로, 최고 JPEG 구성(q=60, 1.15×10⁻⁴) 대비 약 **15.6배 높다**.

**(3) AWGN 강건성:** SNR=0 dB에서도 Semantic dim=256은 91.43% 정확도를 유지한다. dim=64 모델은 압축된 표현으로 인해 낮은 SNR에서 더 민감하게 반응한다(0 dB에서 88.67%).

**(4) 패킷 손실 강건성:** 모든 Semantic 모델은 패킷 손실에 대해 매우 강건하다. dim=256의 경우 손실률 1%~10% 범위에서 정확도 변동이 0.05% 미만으로, 잠재 표현이 태스크 관련 정보를 여러 차원에 분산 저장함을 시사한다.
